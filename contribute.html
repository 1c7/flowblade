Contributing as User

Bug reports

Bug reports by users are an important way to improve the quality of the application.

Bug reports can be filed at the Issues section of the project GitHub page. You need a GitHub account to file bug reports.

Translations

All the translations for Flowblade are community contributed.

If you wish to contribute a new translation or improve on the existing ones, see the instructions here.

Tutorials, articles, demoreel

Flowblade tutorials, articles about it are of interest to the project and there is an Issue page where you can provide information about those.

If you have created a video with Flowblade you would like to show others in demo the reel being created for application, see the related Issue page

Web page

There is an Issue page. for developing this web page.

Contributing as Programmer

Flowblade application

Flowblade is develoment is currently centered on the Flowblade GitHub page. Flowblade is developed in Python.

Contributing to the project is easier then you might think. You do not need in-depth knowledge about technical issues such as codecs, or full understanding of the codebase to contribute.

Take a look at the Issues page.at GitHub for more information.

MLT framework

MLT is the media framework like GStreamer and QuickTime that Flowblade uses. Other applications running on top of MLT include KDEnlive and Shotcut.

If you have solid skills in C and you're looking for something interesting and impactful to work on, then you might be interested in contributing to MLT. For more information, see the MLT web page or write to project lead janne.liljeblad@gmail.com.
